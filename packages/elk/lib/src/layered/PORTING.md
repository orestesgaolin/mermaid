# Faithful ELK Layered port — staged plan

This directory (`lib/src/layered/`) is a **direct, faithful port of the Eclipse
Layout Kernel's `layered` algorithm** (`org.eclipse.elk.alg.layered`, ~50k LOC
of Java), file-by-file, replacing the earlier dagre-substrate approximation in
`../engine/elk_layered.dart`.

**Why:** the dagre-based engine matches ELK on layer assignment but diverges on
within-layer ordering and edge routing (feedback/long edges route through
nodes instead of around them). Matching ELK's *output* requires ELK's *actual*
phases composed on its data model — not dagre tuned to look similar.

**Fidelity gate:** `tool/validation/` runs every graph through real elkjs and
scores agreement. Each stage below must hold or improve those numbers
(currently: 100% layer-assignment, 0 overlaps; within-layer order is the gap
stages 4–6 close).

## Pipeline (ELK `ELKLayered`)

```
P1 cycle breaking → P2 layering → P3 crossing min → P4 node placement → P5 edge routing
```
…interleaved with intermediate processors (long-edge split/join, reversed-edge
restore, port ordering, label dummies, layer constraints, hierarchy).

## Status (stage 8 done)

`layeredLayout()` (in `elk_layered_engine.dart`) composes all 5 phases + 12
intermediate processors in ELK's default slot order, transforms internal RIGHT
flow to the requested direction, and extracts an `ElkResult`. It is now the
**live default behind `ElkLayered.layout()` for flat graphs** (no compound
nodes, no explicit ports, no self-loops, no edge labels, no model-order); other
graphs fall back to the dagre-substrate engine until those features are ported.

**Validated against real elkjs 0.9.3.** `test/elk_vs_elkjs_test.dart` is an
automated parity suite over 7 graphs simple→complex (chain, branch&merge,
dependency, tree, wide layer, cyclic feedback, and ELK's own `images/example`),
defined in `tool/validation/graphs.json` and checked against a committed elkjs
golden (`tool/validation/elkjs_golden.json`, regenerated by
`node tool/validation/run_elkjs.mjs`) so it runs under `dart test` without Node.
Each asserts: **100% layer-assignment agreement**, **0 overlaps**, **no more
geometric edge crossings than elkjs** (all 7 graphs are crossing-free in both
engines), and aspect ratio within ~2× of elkjs. `dart run
tool/validation/compare.dart` also writes side-by-side SVGs (ours | elkjs) per
graph for visual diff. mermaid: 412 tests + 184 corpus green; elk: 31
tests + 7 parity tests green.

**Crossing minimization is now faithfully randomized.** The earlier port did a
single deterministic barycenter pass with RNG tie-breaks replaced by
first-element — that is *not* what ELK does and it produced suboptimal,
tie-broken orders (visible edge crossings on the elk-example). ELK's barycenter
heuristic is deliberately non-deterministic (randomized first layer, ±0.035
per-node perturbation, random sweep direction) so that `compareDifferentRandomizedLayouts`
can explore `thoroughness` (default 7) restarts and keep the order with the
fewest crossings. We now replicate that exactly, using a faithful
`java.util.Random` port (`java_random.dart`) so the result is deterministic and
reproducible across runs. This is what closed the crossings gap.

**Residual — within-layer orientation.** On graphs with a symmetric,
crossing-free optimum (e.g. a simple diamond), the chosen left/right
orientation can be the mirror of elkjs's. Both are valid crossing-free ELK
layouts; the specific orientation is fixed by elkjs's exact RNG *call sequence*,
which we do not reproduce instruction-for-instruction (the plan does not claim
bit-parity). Crossing count and layering match.

## No dagre fallback (done)

The dagre-substrate engine produced a non-ELK layout under the `elk` label,
which was misleading; it has been **removed**. `ElkLayered.layout()` is now a
thin wrapper over `layeredLayout()`. Features not yet ported throw a
descriptive `UnsupportedError` (`unsupportedElkFeature`) instead of silently
approximating — honest by design. `elk_unsupported_test.dart` pins this.

## Hierarchy (done — recursive / SEPARATE_CHILDREN)

Compound nodes are supported: each compound's children lay out in their own
nested `LGraph`, the compound is sized to its content + padding, then the
parent lays out treating it as a sized node; children are returned
parent-relative.

**Edges are assigned by lowest-common-ancestor** (faithful to ELK): an edge
belongs to the LCA graph of its endpoints regardless of which graph it was
*declared* in. `buildGraph` partitions its edge set — an edge whose both
endpoints lie inside a single compound child is pushed down into that child's
nested graph; everything else is owned at the current level. This is what makes
the eclipse-elk README "complexRouter" example lay out as a real dataflow
(`example/readme_repro.dart`): edges declared at the root that connect nodes
deep inside `router`/`channel1`/… now reach their proper nested graph instead of
being dropped.

**Cross-hierarchy edges are split at cluster boundaries** through external-port
dummies (ELK's external-port mechanism). When an edge endpoint lies one level
inside a compound child, the engine mints an external port on the cluster (EAST
for a source/output, WEST for a target/input) and an `EXTERNAL_PORT` dummy inside
the nested graph, constrained to the last/first layer (`lastSeparate`/
`firstSeparate`) so it sits on the border; the real inner node connects to that
dummy. After the nested graph is laid out, the dummy's resolved border position
is copied onto the cluster port, and the per-graph segment polylines are stitched
(source→target) into one edge during extraction. This is what pulls a node toward
the boundary it must exit (e.g. `Counter-q1` is drawn toward `MonitorValue-q1`
outside the `router` cluster) and makes `example/readme_repro.dart` reproduce the
eclipse-elk README "complexRouter" dataflow.

Residual: only **one-level** crossings are split this way; an endpoint nested two
or more levels below the LCA falls back to boundary routing (multi-level recursive
splitting and label positioning on split edges are TODO).

## INCLUDE_CHILDREN faithful port — staged roadmap (in progress)

Goal: replace the recursive **SEPARATE_CHILDREN** approximation (each nested
graph laid out independently, cross-hierarchy edges split through ad-hoc
external ports) with ELK's hierarchy-aware **INCLUDE_CHILDREN** so that
cross-hierarchy edge *ordering* and *clearance* match upstream. This fixes the
three known cross-hierarchy defects:

- **#3** two edges into a node cross right before entering it (the target's
  incoming-port order is decided by the child cluster's own crossing-min, which
  cannot see where the far endpoints sit at the parent level).
- **#4** a cross-cluster edge exits one side then crosses to the other (same,
  on the exit side).
- **#1** a back-edge wrap lane hugs a cluster border (~2px) because the cluster
  is not treated as an obstacle with `edgeNode` clearance.

ELK mechanism (studied from `org.eclipse.elk.alg.layered`):
`CompoundGraphPreprocessor` splits cross-hierarchy edges into outer/inner
segments (external ports + dummies), then the **hierarchy-aware**
`LayerSweepCrossingMinimizer` runs ONE coordinated pass over all graphs in the
hierarchy: per graph `LayerSweepTypeDecider.useBottomUp()` picks bottom-up
(child order drives parent) vs top-down/cross-hierarchical (parent order drives
child); `setPortOrderOnParentGraph()` propagates a child's external-port order
up; `SweepCopy` saves/restores the best node+port orders. Then hierarchical port
processors position the border ports and route the boundary segments with
clearance; `CompoundGraphPostprocessor` reassembles.

Checkpoints (each must compile, keep elk/core/flutter tests green, and not
regress the `tool/elk_oracle.sh` comparison; flat graphs must be byte-identical
so the 7-graph elkjs parity suite + 184 corpus are untouched — hierarchy-aware
code only engages when compound nodes are present):

- **C1** Foundational utilities ported to our LGraph model: `SweepCopy`
  (node+port order save/restore), `GraphInfoHolder` (per-graph hierarchy links +
  aux objects), `LayerSweepTypeDecider.useBottomUp()`. New files under
  `layered/hierarchy/`. Not yet wired → behaviour unchanged.
- **C2** Hierarchy-aware crossing-min orchestration: hoist P3 out of the
  per-graph recursive pipeline; build the graph list, run the coordinated
  typed sweep with `setPortOrderOnParentGraph`, gate to compound graphs only.
  → fixes **#3/#4**.
- **C3** Hierarchical port position/constraint + orthogonal border routing
  (`HierarchicalPort*` processors) so boundary segments keep `edgeNode`
  clearance. → fixes **#1**.
- **C4** (if needed) faithful `CompoundGraphPre/Postprocessor` outer/inner
  segment model, replacing the ad-hoc `_endpointPort`/`_crossSegments` split.

Status: C1 in progress.

## Remaining toward full parity

Still throw `UnsupportedError` until ported: **explicit ports**, **self-loops**
(`SELF_LOOP_*` processors), **edge labels** (`LABEL_DUMMY_*`), **model order**
(`SORT_BY_INPUT_ORDER_OF_MODEL` + `considerModelOrder`), and the
**INCLUDE_CHILDREN** single-pass cross-hierarchy routing refinement. Plus
bit-exact within-layer ordering (currently differs only on crossing-free
tie-breaks vs elkjs's RNG — see validation).

## A real bug this surfaced

`LPort.netFlow` was defined as `outgoing − incoming`; ELK's is `incoming −
outgoing`. That flipped every port side (sources got WEST). Fixed in
`lgraph.dart` to match ELK, so `PortSideProcessor`'s faithful `netFlow < 0 →
EAST` now produces correct sides.

## Notes

- Faithful means same algorithm + same defaults. Two unavoidable deviations,
  documented at each site: (a) RNG tie-breaks (e.g. cycle-breaker max-outflow)
  are made deterministic (first element) since we can't replicate ELK's seeded
  `Random`; (b) EMF/Guava plumbing is replaced with plain Dart.
- Only the **default-config** path + the intermediate processors it needs are
  ported (not all 89 processors — many handle niche options like wrapping,
  unzipping, big-node splitting, comment boxes).
- The dagre engine stays in place and keeps powering `ElkLayered` until this
  port passes the validation gate, then it's swapped out.
